# 数据需求

## 1. 目标说明

本文档用于梳理当前 `ClawGuard` 前端页面所需的数据内容，明确前端展示需要哪些数据、这些数据分别用于哪些模块，以及后端或数据处理链路需要提供什么样的原始信息。

当前项目重点围绕 `OpenClaw 公网暴露监测` 页面展开。

---

## 2. 页面数据需求总览

当前页面主要分为以下几类数据模块：

1. Hero 区
2. 情况分析统计
3. 全球分布
4. 中国境内分布
5. 暴露趋势
6. 版本趋势
7. 暴露服务详情列表

---

## 3. Hero 区数据需求

### 展示位置

页面右侧主内容顶部。

### 展示内容

- 平台标识
- 页面标题
- 页面说明文案
- 资产总量
- 最新快照
- 当前区间

### 所需字段

| 字段             | 类型   | 说明                              |
| ---------------- | ------ | --------------------------------- |
| `total`          | number | 当前总资产数                      |
| `latestSnapshot` | string | 最新快照日期，建议格式 `YYYYMMDD` |
| `sourceDir`      | string | 数据来源目录说明                  |
| `page`           | number | 当前页码                          |
| `pageSize`       | number | 每页大小                          |

### 说明

- `资产总量` 来自总记录数。
- `最新快照` 来自最新数据采集日期。
- `当前区间` 由总数、当前页、分页大小组合得出。

---

## 4. 情况分析统计数据需求

### 展示位置

“情况分析”模块卡片区。

### 所需字段

| 字段              | 类型   | 说明             |
| ----------------- | ------ | ---------------- |
| `historyTotal`    | number | 历史暴露服务总数 |
| `currentExposed`  | number | 当前暴露实例数   |
| `domesticTotal`   | number | 境内暴露总数     |
| `overseasTotal`   | number | 境外暴露总数     |
| `countryCoverage` | number | 覆盖国家/地区数  |
| `cityCount`       | number | 涉及城市数       |
| `vendorCount`     | number | 厂商/版本数      |
| `highRiskCount`   | number | 高风险实例数     |

### 数据来源建议

建议后端基于暴露实例明细表聚合生成。

---

## 5. 全球分布数据需求

### 展示位置

全球地图 + Top 国家/地区排行。

### 所需字段

| 字段                   | 类型   | 说明             |
| ---------------------- | ------ | ---------------- |
| `topCountries`         | array  | 全球国家分布列表 |
| `topCountries[].name`  | string | 国家/地区名称    |
| `topCountries[].value` | number | 对应暴露实例数量 |

### 数据要求

- 名称必须与世界地图 GeoJSON 中的区域名称能匹配。
- 建议按数量降序返回。

---

## 6. 中国境内分布数据需求

### 展示位置

中国地图 + Top 省份排行。

### 所需字段

| 字段                | 类型   | 说明             |
| ------------------- | ------ | ---------------- |
| `provinces`         | array  | 中国省份分布列表 |
| `provinces[].name`  | string | 省份名称         |
| `provinces[].value` | number | 对应暴露实例数量 |

### 数据要求

- 名称必须与中国地图 GeoJSON 中的省份名称匹配。
- 建议按数量降序返回。

---

## 7. 暴露趋势数据需求

### 展示位置

“暴露实例演化趋势”图表。

### 所需字段

| 字段         | 类型     | 说明         |
| ------------ | -------- | ------------ |
| `dates`      | string[] | 日期序列     |
| `daily`      | number[] | 每日暴露数量 |
| `cumulative` | number[] | 累计暴露数量 |
| `newAdded`   | number[] | 每日新增数量 |

### 数据要求

- 四个数组长度必须一致。
- 日期建议按升序排列。

---

## 8. 版本趋势数据需求

### 展示位置

“版本实例演化趋势”图表。

### 所需字段

| 字段                 | 类型     | 说明                     |
| -------------------- | -------- | ------------------------ |
| `dates`              | string[] | 日期序列                 |
| `versions`           | object   | 各版本趋势数据           |
| `versions.<version>` | number[] | 某版本在各日期上的实例数 |

### 数据要求

- 每个版本对应数组长度必须与 `dates` 一致。
- 版本名称建议直接使用前端可展示的文本。

---

## 9. 暴露服务详情列表数据需求

### 展示位置

详情表格、搜索、分页、导出。

### 所需字段

| 字段           | 类型   | 说明               |
| -------------- | ------ | ------------------ |
| `id`           | string | 唯一记录 ID        |
| `ip`           | string | IP 地址            |
| `host`         | string | 主机名             |
| `service`      | string | 端口/服务          |
| `serviceDesc`  | string | 服务描述，可选     |
| `location`     | string | 地理位置           |
| `city`         | string | 城市               |
| `isp`          | string | 运营商             |
| `asn`          | string | ASN，可选          |
| `vendor`       | string | 厂商               |
| `status`       | string | 状态               |
| `scope`        | string | 境内/境外          |
| `version`      | string | 版本号             |
| `risk`         | string | 历史漏洞关联情况   |
| `lastSeen`     | string | 最后发现时间       |
| `lastSnapshot` | string | 最近快照时间，可选 |

### 列表顶层额外字段

| 字段             | 类型   | 说明         |
| ---------------- | ------ | ------------ |
| `total`          | number | 总记录数     |
| `page`           | number | 当前页码     |
| `page_size`      | number | 每页数量     |
| `latestSnapshot` | string | 最新快照日期 |
| `sourceDir`      | string | 数据来源说明 |
| `rows`           | array  | 当前页数据   |

## 10. 登录态与脱敏数据需求

当前前端支持“未登录脱敏、登录后显示完整信息”的模式。

### 需要支持脱敏的字段

- `ip`
- `city`
- `isp`
- `location`

### 建议策略

- 未登录：返回脱敏结果
- 已登录：返回完整结果
- 后端根据权限控制，不建议只依赖前端处理

---

## 11. 数据更新频率需求

建议后端或数据链路明确以下时间属性：

| 字段             | 说明                 |
| ---------------- | -------------------- |
| `latestSnapshot` | 最新快照日期         |
| `lastSeen`       | 单条记录最后发现时间 |
| 趋势日期序列     | 用于展示日变化       |

### 建议

- 至少按天更新一次
- 若支持实时更新，可增加小时级或分钟级快照能力

---

## 12. 最小可用数据集

如果只先支撑“暴露服务详情”页面最核心功能，最少需要：

1. 暴露服务列表数据
2. Hero 统计数据
3. 情况分析统计数据

如果要完整支撑当前页面，则需要：

1. Hero 数据
2. 情况分析统计
3. 全球分布
4. 中国境内分布
5. 暴露趋势
6. 版本趋势
7. 详情表格数据

---

## 13. 推荐数据来源拆分

建议后端或数据处理层至少准备以下数据源：

1. 原始暴露实例明细表
2. 地理位置映射数据
3. 版本聚合统计表
4. 每日快照历史表
5. 风险关联表

这样可以稳定支撑：

- 地图分布
- 趋势图
- 版本图
- 详情检索
- 导出
