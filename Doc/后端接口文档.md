# 后端接口文档

## 1. 文档说明

本文档基于当前前端项目实际使用的数据结构整理，目标是为现有 `ClawGuard` 前端提供可直接对接的后端接口规范。

当前前端主要涉及以下页面与模块：

- 首页
- OpenClaw 公网暴露监测页
- Hero 统计卡片
- 情况分析统计卡片
- 全球分布地图
- 中国境内分布地图
- 暴露趋势图
- 版本趋势图
- 暴露服务详情表格

建议后端统一使用 JSON 返回，字符编码为 `UTF-8`。

建议接口统一前缀：

```text
/api
```

---

## 2. 通用约定

### 2.1 请求头

```http
Content-Type: application/json
Authorization: Bearer <token>
```

说明：

- 当前前端登录还是 mock，尚未真正接入认证接口。
- 如果后续接入真实登录，可在需要脱敏控制的接口上使用 `Authorization`。

### 2.2 通用响应格式

建议统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

字段说明：

- `code`: `0` 表示成功，非 `0` 表示失败
- `message`: 响应说明
- `data`: 实际业务数据

### 2.3 通用错误格式

```json
{
  "code": 40001,
  "message": "invalid params",
  "data": null
}
```

---

## 3. 接口总览

当前建议最少提供 6 个业务接口：

1. `GET /api/exposure/stats`
2. `GET /api/exposure/world-distribution`
3. `GET /api/exposure/china-distribution`
4. `GET /api/exposure/trend`
5. `GET /api/exposure/version-trend`
6. `GET /api/exposure/list`

可选补充接口：

1. `POST /api/auth/login`
2. `POST /api/auth/logout`
3. `GET /api/exposure/export`

---

## 4. 详细接口定义

## 4.1 情况分析统计

### 接口

```http
GET /api/exposure/stats
```

### 用途

用于“情况分析”统计卡片，以及页面顶部 KPI 数据展示。

### 请求参数

无

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "historyTotal": 120000,
    "currentExposed": 116572,
    "domesticTotal": 3200,
    "overseasTotal": 113372,
    "countryCoverage": 87,
    "cityCount": 430,
    "vendorCount": 25,
    "highRiskCount": 418
  }
}
```

### 字段说明

| 字段              | 类型   | 说明             |
| ----------------- | ------ | ---------------- |
| `historyTotal`    | number | 历史暴露服务总数 |
| `currentExposed`  | number | 当前暴露实例数   |
| `domesticTotal`   | number | 境内暴露总数     |
| `overseasTotal`   | number | 境外暴露总数     |
| `countryCoverage` | number | 覆盖国家/地区数  |
| `cityCount`       | number | 涉及城市数       |
| `vendorCount`     | number | 厂商/版本数      |
| `highRiskCount`   | number | 高风险实例数     |

---

## 4.2 全球分布

### 接口

```http
GET /api/exposure/world-distribution
```

### 用途

用于全球暴露实例地图与 Top 国家/地区列表。

### 请求参数

无

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "topCountries": [
      { "name": "United States", "value": 1200 },
      { "name": "Germany", "value": 800 },
      { "name": "Singapore", "value": 760 }
    ]
  }
}
```

### 字段说明

| 字段                   | 类型   | 说明                             |
| ---------------------- | ------ | -------------------------------- |
| `topCountries`         | array  | 国家分布列表                     |
| `topCountries[].name`  | string | 国家或地区名称，需与地图名称匹配 |
| `topCountries[].value` | number | 实例数量                         |

## 4.4 暴露趋势

### 接口

```http
GET /api/exposure/trend
```

### 用途

用于“暴露实例演化趋势”图表。

### 请求参数

可选：

| 参数         | 类型   | 必填 | 说明                        |
| ------------ | ------ | ---- | --------------------------- |
| `start_date` | string | 否   | 开始日期，格式 `YYYY-MM-DD` |
| `end_date`   | string | 否   | 结束日期，格式 `YYYY-MM-DD` |

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "dates": ["2026-03-01", "2026-03-02", "2026-03-03"],
    "daily": [120, 135, 140],
    "cumulative": [10000, 10135, 10275],
    "newAdded": [20, 15, 18]
  }
}
```

### 字段说明

| 字段         | 类型     | 说明         |
| ------------ | -------- | ------------ |
| `dates`      | string[] | 日期序列     |
| `daily`      | number[] | 每日数量     |
| `cumulative` | number[] | 累计数量     |
| `newAdded`   | number[] | 每日新增数量 |

说明：

- `dates`、`daily`、`cumulative`、`newAdded` 长度必须一致。

---

## 4.5 版本趋势

### 接口

```http
GET /api/exposure/version-trend
```

### 用途

用于“版本实例演化趋势”图表。

### 请求参数

可选：

| 参数         | 类型   | 必填 | 说明                            |
| ------------ | ------ | ---- | ------------------------------- |
| `start_date` | string | 否   | 开始日期，格式 `YYYY-MM-DD`     |
| `end_date`   | string | 否   | 结束日期，格式 `YYYY-MM-DD`     |
| `top_n`      | number | 否   | 返回前 N 个版本，默认可返回全部 |

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "dates": ["2026-03-01", "2026-03-02", "2026-03-03"],
    "versions": {
      "v1.0.0": [120, 118, 116],
      "v1.1.0": [80, 95, 102],
      "v2.0.0": [20, 24, 28]
    }
  }
}
```

### 字段说明

| 字段                 | 类型     | 说明                   |
| -------------------- | -------- | ---------------------- |
| `dates`              | string[] | 日期序列               |
| `versions`           | object   | 版本趋势映射           |
| `versions.<version>` | number[] | 某版本在各日期上的数量 |

说明：

- `versions` 下每个版本数组长度必须与 `dates` 一致。

## 4.6 暴露服务详情列表

### 接口

```http
GET /api/exposure/list
```

### 用途

用于详情表格、分页、筛选、导出、Hero 区统计卡片。

### 请求参数

| 参数           | 类型    | 必填 | 说明                                             |
| -------------- | ------- | ---- | ------------------------------------------------ |
| `page`         | number  | 否   | 页码，默认 `1`                                   |
| `page_size`    | number  | 否   | 每页数量，建议支持 `20/50/100`                   |
| `ip`           | string  | 否   | IP 模糊搜索                                      |
| `location`     | string  | 否   | 地理位置模糊搜索                                 |
| `vendor`       | string  | 否   | 厂商模糊搜索                                     |
| `scope`        | string  | 否   | 境内/境外筛选                                    |
| `status`       | string  | 否   | 状态筛选                                         |
| `version`      | string  | 否   | 版本筛选                                         |
| `is_logged_in` | boolean | 否   | 是否返回完整敏感字段，正式环境建议由鉴权自动判断 |

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total": 116572,
    "page": 1,
    "page_size": 20,
    "latestSnapshot": "20260313",
    "sourceDir": "web/clawdbot_alive",
    "rows": [
      {
        "id": "1",
        "ip": "84.52.1.188",
        "host": "-",
        "service": "30023 / OpenClaw",
        "serviceDesc": "OpenClaw service",
        "location": "Slovenia",
        "city": "Ljubljana",
        "isp": "AS3212",
        "asn": "AS3212",
        "vendor": "OpenClaw",
        "status": "Inactive",
        "scope": "海外",
        "version": "-",
        "risk": "未关联历史漏洞",
        "lastSeen": "2026-02-24",
        "lastSnapshot": "2026-02-24"
      }
    ]
  }
}
```

### 字段说明

#### 顶层字段

| 字段             | 类型   | 说明                              |
| ---------------- | ------ | --------------------------------- |
| `total`          | number | 满足筛选条件的总记录数            |
| `page`           | number | 当前页                            |
| `page_size`      | number | 当前每页数量                      |
| `latestSnapshot` | string | 最新快照日期，格式建议 `YYYYMMDD` |
| `sourceDir`      | string | 数据来源目录说明                  |
| `rows`           | array  | 当前页记录                        |

#### rows 字段

| 字段           | 类型   | 说明             |
| -------------- | ------ | ---------------- |
| `id`           | string | 记录唯一 ID      |
| `ip`           | string | IP 地址          |
| `host`         | string | 主机名           |
| `service`      | string | 端口/服务说明    |
| `serviceDesc`  | string | 服务补充说明     |
| `location`     | string | 地理位置         |
| `city`         | string | 城市             |
| `isp`          | string | 运营商           |
| `asn`          | string | ASN              |
| `vendor`       | string | 厂商             |
| `status`       | string | 运行状态         |
| `scope`        | string | 境内/境外        |
| `version`      | string | 版本号           |
| `risk`         | string | 历史漏洞关联说明 |
| `lastSeen`     | string | 最后发现时间     |
| `lastSnapshot` | string | 快照时间         |

### 脱敏建议

未登录时建议后端直接返回脱敏字段，例如：

```json
{
  "ip": "84.*.*.188",
  "city": "***",
  "isp": "***"
}
```

优先建议由后端处理脱敏，而不是交给前端处理。


## 5. 推荐实现方式

## 5.1 推荐分页放在后端

当前前端已经具备分页能力，但正式接口建议后端直接分页：

- 避免一次性返回几十万条数据
- 降低浏览器内存占用
- 提高首屏加载速度
- 便于后端统一筛选与导出

## 5.2 推荐后端承担脱敏逻辑

建议根据登录态、角色或权限，在后端控制：

- IP 是否完整显示
- 城市是否显示
- 运营商是否显示
- 是否允许导出全量数据

## 5.3 推荐保留时间范围查询能力

趋势相关接口建议默认支持：

- `start_date`
- `end_date`

这样后续前端加日期筛选时无需重做接口。

## 7. 最小可用接口集

如果先做最小版本，只需要先提供这 3 个接口：

1. `GET /api/exposure/stats`
2. `GET /api/exposure/list`
3. `GET /api/exposure/trend`

如果要完整覆盖当前页面，建议一次性提供全部 6 个业务接口。
