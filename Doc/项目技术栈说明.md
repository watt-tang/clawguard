# ClawGuard 项目技术栈说明

## 1. 项目概览

ClawGuard 是一个围绕 OpenClaw 生态安全监测构建的全栈项目，当前代码主要集中在 `web` 目录中，采用“前端界面 + 同仓后端 API + MySQL 数据库 + 本地数据处理脚本”的组合方式实现。

项目核心能力包括：

- OpenClaw 公网暴露面展示与查询
- 暴露资产统计、趋势、地域分布、版本分布分析
- 基于 Prisma 的数据库建模与查询
- 基于本地文本快照和 GeoIP 数据的离线数据生成
- 通过 Vite Dev Server 代理后端 API
- 集成 Python 扫描脚本能力，用于 Skill 安全扫描接口

---

## 2. 项目目录结构

```text
clawguard/
├─ Doc/                         # 项目文档
├─ scanner/                     # Python 扫描器与知识库
└─ web/                         # 主应用目录
   ├─ src/                      # React 前端源码
   ├─ server/                   # Express 后端源码
   ├─ prisma/                   # Prisma Schema
   ├─ scripts/                  # 数据生成与资源下载脚本
   ├─ public/                   # 前端静态资源
   ├─ generated/prisma/         # Prisma Client 生成产物
   ├─ clawdbot_alive/           # 原始暴露面快照数据
   ├─ geoip/                    # GeoLite2 数据库文件
   ├─ package.json              # Node 项目配置
   └─ vite.config.js            # Vite 配置
```

---

## 3. 总体架构

项目当前采用单仓库、前后端同目录协作的开发方式：

1. 前端使用 React + Vite 提供控制台式界面。
2. 后端使用 Express 提供 `/api/exposure/*` 接口。
3. 数据层使用 Prisma 连接 MySQL。
4. `scripts/` 和 `server/scripts/` 负责导入、聚合、补充暴露面数据。
5. `vite.config.js` 中通过代理把前端请求转发到本地 API 服务。
6. `scanner/scripts/scan_skill.py` 通过 Vite 自定义插件暴露为 `/api/skill/scan` 接口。

## 4. 前端技术栈

前端位于 `web/src`，主要技术如下：

- React 19
- React DOM 19
- Vite 7
- Tailwind CSS 4（通过 `@tailwindcss/vite` 接入）
- Lucide React 图标库
- ECharts 6 用于图表与地图可视化

### 4.1 前端特点

- 单页应用入口为 `src/main.jsx`
- 主应用组件为 `src/App.jsx`
- 页面模块采用功能目录划分，例如：
  - `features/openclaw-exposure`
  - `features/skill-governance`
- 接口地址统一定义在 `src/config.js`
- 同时保留静态兜底数据路径，API 不可用时可回退到 `public/data` 下的 mock 数据

### 4.2 前端运行方式

在 `web` 目录下执行：

```bash
npm run dev:web
```

该命令会先执行：

```bash
npm run setup
```

然后再启动 Vite 开发服务器。

默认前端开发端口：

```text
5173
```

## 5. 后端技术栈

后端位于 `web/server`，是一个基于 Node.js 的轻量 API 服务，主要技术如下：

- Node.js ESM 模块
- Express 5
- CORS
- dotenv
- Prisma Client

### 5.1 后端职责

后端主要负责：

- 健康检查接口
- 暴露面统计数据查询
- 全球/中国分布数据查询
- 暴露趋势数据查询
- 版本趋势数据查询
- 暴露资产列表查询

### 5.2 主要接口

后端入口文件：`web/server/index.mjs`

核心接口包括：

- `GET /api/health`
- `GET /api/exposure/stats`
- `GET /api/exposure/world-distribution`
- `GET /api/exposure/china-distribution`
- `GET /api/exposure/trend`
- `GET /api/exposure/version-trend`
- `GET /api/exposure/list`

### 5.3 后端运行方式

在 `web` 目录下执行：

```bash
npm run dev:api
```

默认后端端口：

```text
8787
```

端口来源于环境变量：

```env
API_PORT=8787
```

## 6. 数据库技术栈

项目数据库通过 Prisma 管理，Schema 位于：

```text
web/prisma/schema.prisma
```

### 6.1 数据库类型

- MySQL
- Prisma ORM

### 6.2 Prisma 配置

`schema.prisma` 中定义的数据库 provider 为：

```prisma
provider = "mysql"
```

数据库连接通过环境变量 `DATABASE_URL` 提供，例如：

```env
DATABASE_URL="mysql://root:password@127.0.0.1:3306/clawguard?charset=utf8mb4"
```

### 6.3 主要数据模型

当前核心表包括：

- `ExposureSnapshot`
  - 保存每天的快照信息
- `ExposureRecord`
  - 保存每条暴露资产明细
- `ExposureDailyAgg`
  - 保存每日聚合统计结果
- `ExposureVersionDailyAgg`
  - 保存每日版本聚合结果

### 6.4 数据库相关命令

在 `web` 目录下执行：

```bash
npm run db:generate
```

用途：生成 Prisma Client。

```bash
npm run db:push
```

用途：把 Prisma Schema 推送到数据库。

```bash
npm run db:setup
```

用途：一键完成以下步骤：

- 生成 Prisma Client
- 推送数据库结构
- 导入暴露面数据

## 7. 数据处理与初始化流程

这个项目不是单纯的前后端页面，还包含一套本地数据处理流程。

### 7.1 原始数据来源

原始暴露面数据存放在：

```text
web/clawdbot_alive/
```

文件命名模式类似：

```text
server_clawdbot_YYYYMMDD_ip_18789_alive.txt
```

每个文件代表一天的暴露面 IP 快照。

### 7.2 前端静态数据生成

脚本：

```text
web/scripts/generate-exposure-data.mjs
```

作用：

- 读取 `clawdbot_alive` 中的快照文件
- 结合 GeoLite2 数据库补充地理信息和 ASN 信息
- 推断 scope、risk、status、service 等字段
- 生成前端可直接使用的 `public/data/exposure-data.json`

对应命令：

```bash
npm run generate:data
```

### 7.3 版本趋势数据生成

脚本：

```text
web/scripts/generate-version-trend.mjs
```

对应命令：

```bash
npm run generate:version-trend
```

### 7.4 地图数据下载

脚本：

```text
web/scripts/download-geo.mjs
```

作用：

- 下载世界地图 GeoJSON
- 下载中国地图 GeoJSON
- 输出到 `public/data/geo/`

对应命令：

```bash
npm run download:geo
```

### 7.5 后端数据库导入与聚合脚本

位于：

```text
web/server/scripts/
```

包括：

- `import-exposure-data.mjs`
  - 导入暴露面数据到数据库
- `migrate-vendor-to-operator.mjs`
  - 数据字段迁移
- `rebuild-exposure-aggregates.mjs`
  - 重建聚合统计

对应命令：

```bash
npm run db:import:exposure
npm run db:migrate:operator
npm run db:rebuild:agg
```

## 8. 启动方式

### 8.1 本地开发完整启动

推荐在 `web` 目录下执行：

```bash
npm run dev
```

该命令会并行启动：

- 前端 Vite 服务
- 后端 Express API 服务

其中 `dev:web` 在启动前会自动执行 `setup`，也就是：

```bash
npm run generate:data
npm run generate:version-trend
npm run download:geo
```

### 8.2 数据库首次初始化

如果本地 MySQL 数据库尚未初始化，建议执行：

```bash
npm run db:setup
```

### 8.3 推荐启动顺序

1. 启动本地 MySQL，并创建 `clawguard` 数据库
2. 配置 `web/.env`
3. 在 `web` 目录执行 `npm install`
4. 执行 `npm run db:setup`
5. 执行 `npm run dev`

## 9. 环境变量说明

示例文件位于：

```text
web/.env.example
```

当前项目已知环境变量如下：

### 9.1 必填项

```env
DATABASE_URL="mysql://root:password@127.0.0.1:3306/clawguard?charset=utf8mb4"
API_PORT=8787
```

### 9.2 可选项

```env
EXPOSURE_DATA_DIR="D:/clawguard/web/clawdbot_alive"
GEOLITE2_CITY_DB="D:/clawguard/web/geoip/GeoLite2-City.mmdb"
GEOLITE2_ASN_DB="D:/clawguard/web/geoip/GeoLite2-ASN.mmdb"
EXPOSURE_API_PROXY="http://127.0.0.1:8787"
```

补充说明：

- `DATABASE_URL`：Prisma 连接 MySQL 的地址
- `API_PORT`：Express API 监听端口
- `EXPOSURE_DATA_DIR`：暴露面数据目录覆盖路径
- `GEOLITE2_CITY_DB` / `GEOLITE2_ASN_DB`：GeoIP 数据库路径
- `EXPOSURE_API_PROXY`：前端代理的后端 API 地址

## 10. 代理与接口联动方式

在 `vite.config.js` 中定义了开发代理：

- 前端开发服务监听 `5173`
- `/api/exposure` 请求转发到 `http://127.0.0.1:8787`
- 也可以通过 `EXPOSURE_API_PROXY` 覆盖目标地址

这意味着前端开发时可以直接请求：

```text
/api/exposure/...
```

而不需要在前端代码里写死完整域名。

## 11. Scanner 相关技术

除常规前后端外，仓库还包含一个独立的 Python 扫描器模块：

目录：

```text
scanner/
```

主要内容：

- `scanner/scripts/scan_skill.py`
- `scanner/references/threat_knowledge_base.md`

在 `web/vite.config.js` 中，自定义了一个 `skillScanApiPlugin`：

- 接收上传文件或 slug/version 参数
- 调用本地 Python 解释器执行扫描脚本
- 暴露 `/api/skill/scan` 接口

因此，该项目除了 React + Express + MySQL 外，还具备：

- Node 调 Python 的混合处理能力
- 临时文件落盘与 zip 解压能力
- 安全扫描类扩展接口能力
